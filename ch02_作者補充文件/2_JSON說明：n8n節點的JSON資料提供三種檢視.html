Schema 是什麼？
顯示每個欄位的名稱 + 資料型別，不顯示實際值。像是資料的「目錄」，一眼看懂這個節點輸出哪些欄位、是字串還是數字。 最適合：快速確認欄位是否存在、型別是否正確。
Table 是什麼？
把 JSON 攤平成試算表格式，每一列是一筆 item，每一欄是一個欄位。有多筆資料時很容易比較差異。 最適合：確認批次資料、快速掃描多筆結果。
JSON 是什麼？
顯示節點輸出的完整原始結構，包含巢狀物件與陣列。用 {{ $json["欄位名"] }} 就能在下一個節點引用任何值。 最適合：撰寫 expression、除錯、確認巢狀結構。
如何在下一個節點引用資料？
取得當前 item 的欄位
{{ $json["name"] }} {{ $json["user"]["email"] }}
取得指定節點的輸出
{{ $node["HTTP Request"].json["id"] }} {{ $items("Edit Fields")[0].json["status"] }}
提示：在任何欄位輸入 {{ 就會開啟 expression 編輯器，可以直接點選 Schema 中的欄位自動填入路徑。

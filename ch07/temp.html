Webhook URL 的需求情況

發送訊息時不需要 HTTPS Webhook URL

  • 如果您只是要「主動發送」LINE 訊息給用戶（Push Message），不需要設定 Webhook URL

  • 您只需要在 LINE Developers Console 中取得 Channel Access Token 即可

  • n8n 的 LINE 節點可以直接使用這個 Token 來呼叫 LINE Messaging API

何時需要 Webhook URL

  • 只有當您要「接收」來自 LINE 用戶的訊息時，才需要設定 Webhook URL

  • 這種情況下，LINE 平台需要一個公開的 HTTPS 網址來推送事件給您

本機開發的解決方案

如果未來需要接收 LINE 訊息，您可以使用以下方法：

  1. ngrok - 建立臨時的公開 HTTPS 網址

ngrok http 5678 # n8n 預設端口

  1. Cloudflare Tunnel - 免費且穩定的隧道服務

  2. localtunnel - 另一個簡單的隧道工具

純發送訊息的設定步驟

  1. 在 LINE Developers Console 建立 Messaging API Channel

  2. 取得 Channel Access Token

  3. 在 n8n 中使用 HTTP Request 節點或 LINE 節點

  4. 設定 Token 後即可發送訊息

您目前如果只需要發送訊息，完全不需要擔心 Webhook URL 的問題！

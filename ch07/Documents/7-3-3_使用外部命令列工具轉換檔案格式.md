# 7-3-3 使用外部命令列工具轉換檔案格式

在 n8n 中，**Convert to File** 節點雖然可以將資料轉換成多種格式，但並不支援輸出 PDF 或 DOCX 格式。因此，我們需要借助兩個外部工具程式來完成這些格式的轉換：

| 工具 | 路徑 | 用途 |
|------|------|------|
| **Pandoc** | `C:\n8n-data\tools\pandoc\pandoc` | 轉換為 DOCX（及多種文件格式） |
| **wkhtmltopdf** | `C:\n8n-data\tools\wkhtmltox\bin\wkhtmltopdf.exe` | 將 HTML 轉換為 PDF |

> 本節使用的是**可攜式版本（Portable）**，無需安裝，直接解壓縮至 `C:\n8n-data\tools` 即可使用。

---

## 工具介紹

### Pandoc
Pandoc 是一款萬用文件格式轉換工具，支援 Markdown、HTML、DOCX、PDF 等數十種格式的互相轉換。在本節中，我們主要用它將 **Markdown 或 HTML 轉換為 DOCX**。

### wkhtmltopdf
wkhtmltopdf 是一款將 **HTML 轉換為 PDF** 的命令列工具，使用 WebKit 渲染引擎，可完整保留網頁的樣式與排版。

---

## 在 n8n 中使用 Execute Command 節點呼叫工具

### 方法一：將 HTML 轉換為 PDF（使用 wkhtmltopdf）

**步驟：**
1. 在 workflow 中新增 **Execute Command** 節點
2. 在 Command 欄位輸入以下指令：

```
"C:\n8n-data\tools\wkhtmltox\bin\wkhtmltopdf.exe" "{{ $json.inputPath }}" "{{ $json.outputPath }}"
```

**實際範例（固定路徑）：**
```
"C:\n8n-data\tools\wkhtmltox\bin\wkhtmltopdf.exe" "C:\n8n-data\input\report.html" "C:\n8n-data\output\report.pdf"
```

**常用參數：**
```
"C:\n8n-data\tools\wkhtmltox\bin\wkhtmltopdf.exe" --page-size A4 --encoding utf-8 "input.html" "output.pdf"
```

| 參數 | 說明 |
|------|------|
| `--page-size A4` | 設定頁面尺寸 |
| `--encoding utf-8` | 指定編碼，避免中文亂碼 |
| `--margin-top 10mm` | 設定上邊距 |
| `--orientation Landscape` | 設定橫向列印 |

---

### 方法二：將 Markdown / HTML 轉換為 DOCX（使用 Pandoc）

**步驟：**
1. 在 workflow 中新增 **Execute Command** 節點
2. 在 Command 欄位輸入以下指令：

```
"C:\n8n-data\tools\pandoc\pandoc" "{{ $json.inputPath }}" -o "{{ $json.outputPath }}"
```

**實際範例（Markdown 轉 DOCX）：**
```
"C:\n8n-data\tools\pandoc\pandoc" "C:\n8n-data\input\report.md" -o "C:\n8n-data\output\report.docx"
```

**實際範例（HTML 轉 DOCX）：**
```
"C:\n8n-data\tools\pandoc\pandoc" "C:\n8n-data\input\report.html" -o "C:\n8n-data\output\report.docx"
```

**常用參數：**
```
"C:\n8n-data\tools\pandoc\pandoc" input.md -o output.docx --reference-doc=template.docx
```

| 參數 | 說明 |
|------|------|
| `-o` | 指定輸出檔案（副檔名決定格式） |
| `--reference-doc` | 套用自訂 DOCX 樣板 |
| `-f markdown` | 明確指定輸入格式 |
| `-t docx` | 明確指定輸出格式 |

## 完整 n8n Workflow 範例

以下示範一個完整的自動化流程：**監控資料夾 → 偵測 HTML 檔案 → 轉換為 PDF**

```
Local File Trigger
      ↓
Edit Fields（取得 path、name、副檔名）
      ↓
Switch（判斷副檔名）
      ├── html → Execute Command（wkhtmltopdf 轉 PDF）
      └── md   → Execute Command（Pandoc 轉 DOCX）
```

### Edit Fields 節點設定

| 欄位名稱 | 值（Expression） |
|---------|----------------|
| `path` | `{{ $json.path }}` |
| `name` | `{{ $json.path.split('\\').last() }}` |
| `ext` | `{{ $json.path.split('.').last().toLowerCase() }}` |
| `nameNoExt` | `{{ $json.path.split('\\').last().split('.')[0] }}` |

### Execute Command 節點（HTML → PDF）

```
"C:\n8n-data\tools\wkhtmltox\bin\wkhtmltopdf.exe" --encoding utf-8 "{{ $json.path }}" "C:\n8n-data\output\{{ $json.nameNoExt }}.pdf"
```

### Execute Command 節點（MD → DOCX）

```
"C:\n8n-data\tools\pandoc\pandoc" "{{ $json.path }}" -o "C:\n8n-data\output\{{ $json.nameNoExt }}.docx"
```

## 注意事項

1. **路徑含空格時**，務必用雙引號 `" "` 包住路徑
2. **中文路徑**可能導致部分工具出錯，建議路徑使用英文
3. **wkhtmltopdf 轉換中文**時，需加上 `--encoding utf-8` 參數
4. **Pandoc 的輸入格式**由副檔名自動判斷，也可用 `-f` 明確指定
5. 可攜式版本**不需設定環境變數**，但指令中必須使用**完整路徑**呼叫工具
